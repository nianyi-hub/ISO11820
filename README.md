
##  项目背景

在建材防火检测实验室中，需要将建筑材料样品放入加热炉，加热至 **750°C** 并持续记录 **60 分钟**的温度变化，依据 ISO 11820 标准判定材料是否属于"不燃材料"。

本系统在**无真实硬件**的条件下，通过**软件仿真**完整复现试验流程：

```
样品登记 → 炉体升温(仿真) → 温度稳定判定 → 60分钟数据记录 → 报告自动生成
```

| 属性         | 说明                                      |
| ------------ | ----------------------------------------- |
| 项目类型     | Windows 桌面应用（WinForms）              |
| 开发语言     | C# / .NET 8                               |
| 数据库       | SQLite（本地文件，零配置）                |
| 运行环境     | Windows 10 / 11，无需联网，无需硬件       |
| 仿真通道数   | 5 路温度通道 + 1 路校准通道               |
| 导出格式     | CSV / Excel（含图表） / PDF               |

##  系统架构

```
┌─────────────────────────────────────────────────────┐
│                    表示层 (WinForms)                  │
│  LoginForm  │  MainForm  │  NewTestForm  │  ...      │
├─────────────────────────────────────────────────────┤
│                    核心业务层                         │
│              TestMasterController                    │
│         (状态机 · 流程编排 · 事件广播)                │
├───────────────────────┬─────────────────────────────┤
│      服务层            │         数据层               │
│  DaqWorker            │       DbHelper              │
│  SensorSimulator      │       (SQLite CRUD)         │
│  ExportService        │                             │
├───────────────────────┴─────────────────────────────┤
│                    基础设施层                         │
│  AppContext(单例)  │  ConfigManager  │  Logger       │
└─────────────────────────────────────────────────────┘
```

**设计原则：**

- **单向依赖** — 上层依赖下层，下层通过 Event 向上通知，绝不反向调用
- **UI 线程安全** — 后台线程数据统一通过 `Invoke` 调度到 UI 线程
- **仿真/硬件双模** — 通过 `EnableSimulation` 配置开关，一套接口两种实现
- **状态机驱动** — 5 状态严格流转，按钮可用性由状态机决定

##  仿真引擎

### 温度通道

系统仿真 **5 个温度通道**，全部由 `SensorSimulator` 自动生成，不依赖任何真实硬件：

| 通道号 | 代号  | 中文名称     | 仿真行为                                       |
| ------ | ----- | ------------ | ---------------------------------------------- |
| 0      | TF1   | 炉内温度 1   | 主加热炉温度；升温阶段线性上升，稳定阶段钳位 750°C |
| 1      | TF2   | 炉内温度 2   | 副加热炉温度；与 TF1 同步但有独立随机噪声       |
| 2      | TS    | 样品表面温度 | 记录阶段指数趋近炉温×0.95，非记录阶段≈炉温×0.3 |
| 3      | TC    | 样品中心温度 | 记录阶段指数趋近炉温×0.85（比 TS 慢），非记录阶段≈炉温×0.25 |
| 16     | TCal  | 校准温度     | = TF1 + 随机波动×2，仅数值显示不绘制曲线       |

### 仿真参数

| 参数                     | 默认值    | 说明                             |
| ------------------------ | --------- | -------------------------------- |
| `InitialFurnaceTemp`     | 720.0 °C  | 仿真起始炉温（便于快速演示）     |
| `TargetFurnaceTemp`      | 750.0 °C  | 目标炉温                         |
| `HeatingRatePerSecond`   | 40.0 °C/s | 升温速率                         |
| `TempFluctuation`        | 0.5 °C    | 温度随机波动幅度                 |
| `StableThreshold`        | 3.0 °C    | 稳定判定窗口（目标温度 ± 阈值）  |

### 状态机

```
                    ┌───────────┐
          开始升温   │           │   温度稳定
    ┌──────────────▶│ Preparing │──────────────┐
    │               │  (升温中)  │              │
    │               │           │              │
    │               └─────┬─────┘              │
    │     停止加热        │ 温度跌出范围       │
    │                     │                    ▼
┌───┴───┐            ┌───┴───┐            ┌──────┐
│  Idle  │            │ Ready  │  开始记录   │Ready │
│ (空闲)  │◀───────────│ (就绪)  │────────────▶│(就绪) │
└───────┘   停止加热    └───────┘            └──┬───┘
    ▲                                          │
    │           ┌──────────┐                   │ 开始记录
    │           │ Complete  │◀──────────────────┘
    └───────────│  (完成)    │   到达时长/终止条件
      保存后清空  │           │
                └──────────┘
```

>  **设计要点**：Complete 完成后自动回到 Preparing（保持炉温），避免冷却后再升温的等待。保存试验记录后清空当前试验缓存，可以新建下一次试验。

##  数据库设计

系统使用 SQLite 本地数据库，共 **6 张表**：

| 表名                    | 记录数  | 说明                             |
| ----------------------- | ------- | -------------------------------- |
| `operators`             | 2       | 操作员账号（管理员 + 试验员）     |
| `apparatus`             | 1       | 试验设备信息                     |
| `productmaster`         | N       | 样品信息                         |
| `testmaster`            | N       |  试验记录（核心表，50+ 字段）   |
| `sensors`               | 17      | 传感器通道配置                   |
| `CalibrationRecords`    | N       | 设备校准历史记录                 |

>  完整表结构及建表 SQL 见 [DB-数据库设计.md](DB-数据库设计.md)

### 温度时序数据

逐秒温度数据**不存入数据库**，而是以 CSV 文件形式存储：

```
{BaseDirectory}\TestData\{productId}\{testId}\sensor_data.csv
```

| 列名           | 单位   | 说明         |
| -------------- | ------ | ------------ |
| `Time`         | 秒     | 记录时间     |
| `Temp1`        | °C     | 炉温 1       |
| `Temp2`        | °C     | 炉温 2       |
| `TempSurface`  | °C     | 表面温度     |
| `TempCenter`   | °C     | 中心温度     |
| `TempCalibration` | °C  | 校准温度     |

##  配置说明

主配置文件位于 `src/appsettings.json`，关键配置项如下：

```jsonc
{
  "Database": {
    "Provider": "Sqlite",
    "SqlitePath": "Data\\ISO11820.db"           // 数据库文件路径
  },
  "Simulation": {
    "EnableSimulation": true,                    // true=仿真模式, false=硬件模式
    "InitialFurnaceTemp": 720.0,                 // 初始炉温 (便于快速演示)
    "TargetFurnaceTemp": 750.0,                  // 目标炉温
    "HeatingRatePerSecond": 40.0,                // 升温速率
    "TempFluctuation": 0.5,                      // 温度波动幅度
    "StableThreshold": 3.0                       // 稳定判定阈值
  },
  "FileStorage": {
    "BaseDirectory": "D:\\ISO11820",             // 数据存储根目录
    "TestDataDirectory": "D:\\ISO11820\\TestData" // 测试数据目录
  },
  "Report": {
    "OutputDirectory": "D:\\ISO11820\\Reports",  // 报告输出目录
    "EnablePdfExport": true                      // 是否启用 PDF 导出
  }
}
```
